# Mission: Agentic Skills Best Practices

## Why I'm Learning This

I build production-grade AI agents (CLI + workflows) and create reusable developer tools. I need a systematic methodology for designing agentic skills files that are modular, maintainable, and performant — moving beyond ad-hoc prompt templates to structured, composable skill architectures that scale across projects and teams.

## What Success Looks Like

- I can design skill files with clean architecture (metadata, interfaces, logic, workflows, configuration)
- I apply the Single Responsibility Principle to skill scope definition
- My skills integrate cleanly with MCP and tool ecosystems
- I version and maintain skills sustainably
- I avoid common anti-patterns (monolithic skills, hard-coded config, missing error handling)
- I can evaluate and test skills systematically
- I understand governance patterns for deploying agents at enterprise scale

## Current Level

Experienced builder with hands-on agent development (Go agentic loops, Anthropic API, Kiro CLI skills). Missing: formal methodology for skill file design, enterprise governance patterns, and systematic testing/validation approaches.
