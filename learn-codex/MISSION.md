# Mission: Use Codex Alongside Existing Coding Agents

## Why
Use Codex to complement an established Kiro CLI and Claude workflow, gaining independent implementation and review capacity without rebuilding familiar agent habits from scratch.

## Success looks like
- Choose the right Codex surface for a task.
- Delegate a bounded change with explicit verification criteria.
- Use Codex as an independent reviewer of work produced elsewhere.
- Adopt Codex-specific capabilities only when they improve the existing workflow.

## Constraints
- Assume familiarity with coding agents, repositories, terminal workflows, and prompt-driven development.
- Prefer short, practical lessons over introductory AI explanations.

## Out of scope
- Re-teaching basic prompting or software development.
- Replacing Kiro CLI or Claude wholesale.
