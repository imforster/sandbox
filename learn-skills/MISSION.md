# Mission: Building Better Skills & Agents

## Why

I maintain 30+ agent skills across Kiro CLI and publish courses on agentic patterns. My skills work — but I want them to be *predictable*. I want the vocabulary to reason about why one skill steers well and another drifts, and I want a discipline for evolving a skill set without accumulating sediment.

## Success looks like

- A tighter, more predictable personal skill set — measured by fewer corrections per session
- A published methodology I can teach others (clients, course learners, blog readers)
- The ability to audit any skill file and immediately identify failure modes (sprawl, negation, no-ops, premature completion)
- A clear lifecycle model for when to draft, promote, rename, deprecate, or kill a skill

## Constraints

- Learning by doing — refactoring real skills from my own repo, not toy examples
- Building on two primary sources: Matt Pocock's skills trail-map (6 months of evolution data) and my own agents-at-scale course material
- Each lesson should produce an artifact I can use immediately (a refactored skill, a checklist, a reference card)

## Out of scope

- Building a skills framework/library (that's a separate project)
- Teaching others how to install or use skills (that's agents-at-scale)
- The mechanics of specific agent runtimes (Strands, Codex, Claude Code internals)
